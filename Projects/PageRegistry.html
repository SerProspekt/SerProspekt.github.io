    • Page Registry

    PageRegistry fundamentally is a data manager and a type identifier; The data it manages is data shared between DLLs. It's hyper-fast, as bare as it gets, it can act as a forwarder for custom containers for frictionless container swapping to manage the data however you'd like. Type identifying works simply by providing IDs for each DLL and every type registered, so you can query and retrieve type info i.e. rudimentary reflection (very fast no RTTI). Everything is automated, the user only has to add/get/remove items from the registry. PageRegistry also provides template functor callbacks, a zero-cost-abstraction OnRegister/OnDeregister/OnReorder/OnReset.

    Now if you can register/deregister types, the only logical next step would be DLL reloading.

    PageRegistry is built for this, it automatically figures out types registered to specific DLLs and unloads them when the DLL gets removed (This can be disabled for full manual control) The automated nature of PageRegistry is a core principle of Page. Types are automatically registered as well when used. There is no manual registration required. This is achieved with highly esoteric C++ tricks allowing for all of this, template madness, C++ obscure trickery and plain old function pointers.
